스크린샷 기반 지식 카드는 생산 속도가 빠른 대신 품질 편차가 큽니다.  
특히 주간 목표를 숫자로만 두면, 나중에 다시 읽기 어려운 카드가 쌓입니다.

그래서 주간 실험에서는 개수와 별도로 **품질 루브릭**을 같이 운영해야 합니다.

## 1) 카드 1장의 최소 완성 조건

아래 3가지를 모두 만족해야 “완료”로 봅니다.

- 핵심 요약 3줄 이내
- 다음 액션 1개
- 출처 맥락 1줄(어디서 나온 정보인지)

요약만 있고 액션이 없으면 기록은 남아도 실행이 이어지지 않습니다.

## 2) 입력 품질 지표를 같이 본다

생성 결과만 보면 원인을 놓칩니다. 입력 품질도 같이 봐야 합니다.

- OCR 텍스트 길이
- 이미지 입력 유무
- 요약 라인 수

입력이 빈약하면 모델을 바꿔도 결과는 개선되지 않습니다.

## 3) 주간 목표는 2축으로 잡는다

- 수량 목표: 주 15장
- 품질 목표: 완성 조건 충족률 80%+

수량만 높이면 노이즈가 쌓이고, 품질만 높이면 운영 속도가 떨어집니다.  
두 축을 같이 봐야 실험이 유지됩니다.

## 4) 실패 사례는 템플릿으로 재사용한다

실패 카드를 지우기보다 실패 패턴으로 남깁니다.

- 입력 없음
- 요약 과잉
- 액션 부재

패턴이 누적되면 다음 주 프롬프트와 체크리스트가 빠르게 개선됩니다.

## 근거: 지식 카드의 가치는 “회수율”로 결정된다

나중에 다시 꺼내 쓸 수 없는 카드는 저장 비용만 남깁니다.

- 액션 없는 카드는 실행 전환율이 낮습니다.
- 맥락 없는 카드는 검색 정확도가 떨어집니다.
- 품질 기준 없는 실험은 주차가 지날수록 유지가 무너집니다.

## 실행 방법: 스크린샷 카드 주간 운영

1. 카드 템플릿에 3줄 요약/액션/출처를 고정한다.
2. 생성 로그에서 입력 품질 지표를 같이 저장한다.
3. 매일 1회, 완성 조건 충족률만 빠르게 점검한다.
4. 주간 리뷰에서 실패 패턴 3개를 추출한다.
5. 다음 주 프롬프트와 체크리스트를 해당 패턴 기준으로 수정한다.

## 오늘의 메모

카드는 많이 만드는 것보다  
나중에 다시 쓰일 형태로 만드는 게 먼저입니다.
