Raycast에 명령이 많아질수록 역설적으로 실행률은 떨어집니다.  
무엇을 먼저 실행할지 매번 고민하게 되기 때문입니다.

옵시디언 운영 가이드를 기준으로 정리한 결과, 하루 운영은 아래 5개 스택이 가장 안정적이었습니다.

## 1) 시작 명령은 하나만 둔다

- `System Daily Check`

아침 시작 명령을 하나로 고정하면, 점검 누락이 급격히 줄어듭니다.

## 2) 상태 확인과 진단을 분리한다

- 빠른 확인: `Dashboard`
- 상세 진단: `Health Check`

두 명령의 역할이 섞이면 같은 정보를 두 번 보고도 결정을 못 내립니다.

## 3) 정리 작업은 반드시 Report 선행

- `Nightly Cleanup Report`
- `Nightly Cleanup Apply`

파괴적 작업은 “보고서 확인 → 적용” 2단계를 강제해야 안전합니다.

## 4) 지표 확인은 마지막에 붙인다

- `Knowledge KPI`

운영 상태가 안정된 뒤에 지표를 보는 흐름이 실용적입니다.  
지표를 먼저 보면 액션이 아니라 분석으로 하루를 시작하게 됩니다.

## 근거: 운영 콘솔의 품질은 명령 수가 아니라 순서에서 나온다

실패율이 높은 날을 보면 기능 부족보다 **실행 순서 혼선**이 원인인 경우가 많습니다.

- 시작 명령이 여러 개면 루틴 진입이 느려집니다.
- Report/Apply가 섞이면 실수 비용이 커집니다.
- 진단 명령을 먼저 실행하면 과분석으로 이어집니다.

## 실행 방법: 3분 운영 루틴

1. System Daily Check 실행.
2. 이상 징후가 있으면 Dashboard로 범위를 확인.
3. 필요 시 Health Check로 원인을 진단.
4. 정리 필요가 있을 때만 Report를 확인.
5. Apply는 보고서 확인 후에만 실행.

## 오늘의 메모

운영 커맨드는 많을수록 좋은 게 아니라  
순서가 고정될수록 강해집니다.
