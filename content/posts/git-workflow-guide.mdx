## 브랜치 전략이 중요한 이유

혼자 개발할 때는 main 브랜치 하나로도 충분하지만, 팀 단위 개발에서는 체계적인 브랜치 전략이 필수입니다. 명확한 전략 없이는 코드 충돌, 불안정한 릴리스, 비효율적인 코드 리뷰가 반복됩니다.

## GitHub Flow

가장 간단하면서도 효과적인 전략입니다:

```
main ─────────────────────────────────
  └── feature/login ─── PR ─── merge
  └── fix/header-bug ── PR ─── merge
```

규칙:
1. `main` 브랜치는 항상 배포 가능한 상태 유지
2. 새 작업은 `main`에서 브랜치 생성
3. 작업 완료 후 Pull Request 생성
4. 코드 리뷰 후 `main`에 merge

```bash
# 새 기능 브랜치 생성
git checkout -b feature/user-auth

# 작업 완료 후 push
git push -u origin feature/user-auth

# GitHub에서 PR 생성 후 merge
```

## 브랜치 네이밍 컨벤션

```
feature/기능명   - 새로운 기능 개발
fix/버그명       - 버그 수정
hotfix/긴급수정  - 프로덕션 긴급 수정
refactor/대상    - 코드 리팩토링
docs/문서명      - 문서 작업
```

예시:
```
feature/user-authentication
fix/login-redirect-error
hotfix/payment-calculation
refactor/api-response-handler
```

## 커밋 메시지 컨벤션

Conventional Commits 형식을 따릅니다:

```
<타입>(<범위>): <설명>

feat: 사용자 인증 기능 추가
fix: 로그인 리다이렉트 오류 수정
docs: API 문서 업데이트
refactor: 유틸리티 함수 구조 개선
chore: 의존성 업데이트
style: 코드 포맷팅
test: 인증 모듈 테스트 추가
```

좋은 커밋 메시지의 원칙:
- 명령형으로 작성 ("추가한다" → "추가")
- 50자 이내로 간결하게
- 무엇을 했는지보다 **왜** 했는지 설명

## Pull Request 작성법

PR은 코드 리뷰의 기본 단위입니다:

```markdown
## 변경 사항
- 사용자 로그인/로그아웃 기능 구현
- JWT 토큰 기반 인증 미들웨어 추가

## 관련 이슈
- closes #42

## 테스트
- [x] 로그인 성공/실패 케이스 테스트
- [x] 토큰 만료 시 리다이렉트 확인
- [x] 로그아웃 후 보호된 페이지 접근 차단 확인

## 스크린샷
(UI 변경 시 첨부)
```

## 유용한 Git 명령어

### 작업 임시 저장

```bash
# 현재 변경사항 임시 저장
git stash

# 임시 저장한 변경사항 복원
git stash pop

# 저장 목록 확인
git stash list
```

### 커밋 히스토리 관리

```bash
# 마지막 커밋 메시지 수정
git commit --amend -m "수정된 메시지"

# 특정 파일만 스테이징
git add -p

# 커밋 로그 그래프 확인
git log --oneline --graph --all
```

### 실수 되돌리기

```bash
# 스테이징 취소
git restore --staged 파일명

# 작업 내용 되돌리기
git restore 파일명

# 특정 커밋으로 되돌리기 (새 커밋 생성)
git revert 커밋해시
```

## .gitignore 작성

프로젝트에 불필요한 파일을 추적하지 않습니다:

```
# 의존성
node_modules/

# 빌드 결과물
.next/
dist/

# 환경 변수
.env
.env.local

# IDE 설정
.vscode/
.idea/

# OS 파일
.DS_Store
Thumbs.db
```

## 마무리

Git은 도구일 뿐이지만, 어떻게 사용하느냐에 따라 팀의 생산성이 크게 달라집니다. 명확한 브랜치 전략, 일관된 커밋 메시지, 충실한 PR 작성을 습관화하면 협업의 질이 높아지고 코드베이스의 건강도가 유지됩니다.
