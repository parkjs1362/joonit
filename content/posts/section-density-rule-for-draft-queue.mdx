초안을 쓸 때 가장 자주 생기는 문제는 분량 부족이 아니라 **실행 항목 부족**입니다.  
저는 이를 막기 위해 "각 섹션 2개 이상" 규칙을 고정하고, 초안 단계에서 바로 검증합니다.

## 1) 섹션 밀도 규칙을 먼저 정의한다

글을 다 쓴 뒤 보정하는 방식보다, 시작 전에 기준을 잠가두는 쪽이 안정적입니다.

- 모든 본문 섹션에 실행 문장 또는 체크 항목을 최소 2개 넣습니다.
- 설명 문단만 있는 섹션은 발행 후보에서 제외합니다.
- 섹션 길이보다 "행동 가능 문장 수"를 우선 지표로 둡니다.

### 통과 기준

- 섹션 수 3개 이상
- 섹션별 실행 항목 2개 이상
- 체크리스트 항목 4개 이상

## 2) 초안 작성 순서를 바꾼다

문장부터 쓰면 설명 위주로 흐르기 쉬워서, 저는 항목부터 고정합니다.

1. 제목 아래에 핵심 섹션 3개를 먼저 확정합니다.
2. 각 섹션에 실행 항목 2개를 먼저 작성합니다.
3. 그 다음에 연결 문단을 붙여 자연스럽게 읽히게 만듭니다.
4. 마지막으로 중복 문장을 2개 이하로 줄입니다.

### 작성 중 점검 질문

- 이 섹션을 읽고 독자가 바로 할 수 있는 행동이 2개 이상인가
- 측정 가능한 기준(횟수, 기간, 비율)이 최소 1개 포함됐는가

## 3) 리뷰 단계에서 제거 기준을 명확히 둔다

추가보다 제거가 품질을 올립니다. 제거 규칙을 미리 두면 리뷰 시간이 줄어듭니다.

- 같은 의미를 반복하는 문단은 1개만 남기고 삭제합니다.
- 배경 설명이 길고 행동 항목이 없는 부분은 별도 메모로 분리합니다.
- 내부 전용 정보나 민감한 표현은 공개 문장으로 재작성합니다.

### 실패 복구 규칙

- 섹션 하나라도 항목 2개 미만이면 발행 큐에 넣지 않습니다.
- 2회 연속 미통과 시 글 길이를 30% 축소하고 핵심 항목부터 다시 씁니다.

## 4) 주간 운영에 붙이는 방법

규칙은 한 번 정하는 것보다 주간 루틴에 붙여야 유지됩니다.

- 월~수: 초안 작성(섹션 항목 2개 규칙만 확인)
- 목: 구조/신뢰 점검(숫자 기준, 민감정보 제거)
- 금: 통과 글만 큐로 이동
- 토: 미통과 글 1편만 재작업

### 최소 지표

- 주간 통과 글 수
- 섹션 항목 누락 건수
- 재작업 후 통과율

## 5) 바로 적용 체크리스트

- [ ] 본문 섹션이 3개 이상인가
- [ ] 각 섹션에 실행 항목이 2개 이상인가
- [ ] 측정 가능한 기준이 2개 이상인가
- [ ] 민감정보/내부 전용 표현이 제거됐는가
- [ ] 이번 주 통과 글이 2편 이상 확보됐는가

섹션 밀도 규칙은 글을 더 길게 만드는 도구가 아니라, 발행 가능한 글을 더 자주 만드는 운영 기준입니다.
