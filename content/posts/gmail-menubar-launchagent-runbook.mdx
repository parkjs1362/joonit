메뉴바 앱은 만들어 놓고 끝나는 도구가 아닙니다.  
실제로 오래 쓰려면 기능보다 **백그라운드 실행 안정성**이 먼저 확보되어야 합니다.

이 글은 Gmail 필터 메뉴바 앱을 운영하면서 효과가 있었던 LaunchAgent 중심 점검 루틴을 정리한 내용입니다.

## 1) 실행 경로를 먼저 고정한다

메뉴바 앱 문제의 절반은 실행 경로 불일치에서 발생합니다.  
Python 런타임 경로, 실행 파일 위치, 로그 파일 경로를 먼저 고정해야 재현 가능한 디버깅이 가능합니다.

- 런타임 경로 고정
- plist 경로 명시
- 로그 파일 단일화

경로가 고정되면 장애가 나도 원인 추적 속도가 빨라집니다.

## 2) launchctl 명령은 세트로 쓴다

운영 중에는 시작 명령 하나만 알면 부족합니다.  
상태 확인과 재시작, 로그 확인 명령을 항상 같이 묶어야 합니다.

- kickstart
- list/print
- 로그 tail

특히 재시작 후 상태 확인을 생략하면 “실행된 줄 알았는데 미실행” 상태를 놓치기 쉽습니다.

## 3) 토글 기능은 실패 기준을 UI에 드러낸다

자동실행 토글이나 일일 실행 토글은 성공/실패가 눈에 보이지 않으면 신뢰를 잃습니다.  
실패 시 아이콘 상태와 알림 문구를 명확히 보여줘야 사용자 행동이 빨라집니다.

## 4) 일일 스케줄은 기능보다 관측을 먼저 붙인다

스케줄 실행은 “됐겠지”가 가장 위험합니다.  
하루 실행 횟수, 처리 건수, 마지막 실행 시각을 남겨두면 이상 징후를 바로 잡을 수 있습니다.

## 체크리스트

- [ ] 실행/로그/설정 경로가 문서에 고정돼 있는가
- [ ] 시작/상태/재시작 명령이 하나의 런북으로 정리돼 있는가
- [ ] 토글 실패 시 UI 피드백이 즉시 보이는가
- [ ] 일일 실행 통계를 확인할 수 있는가
