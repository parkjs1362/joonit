## 유틸리티 타입이란?

TypeScript는 타입 변환을 쉽게 하기 위한 여러 내장 유틸리티 타입을 제공합니다. 이들을 잘 활용하면 중복 없이 타입을 유연하게 관리할 수 있습니다.

## 자주 사용하는 유틸리티 타입

### Partial&lt;T&gt;

모든 프로퍼티를 선택적으로 만듭니다. API의 업데이트 요청에서 특히 유용합니다:

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  avatar: string;
}

// 업데이트 시에는 일부 필드만 보낼 수 있음
function updateUser(id: number, updates: Partial<User>) {
  // updates는 { name?: string; email?: string; avatar?: string; }
  return fetch(`/api/users/${id}`, {
    method: 'PATCH',
    body: JSON.stringify(updates),
  });
}

// 이름만 변경
updateUser(1, { name: '새 이름' });
```

### Pick&lt;T, K&gt;

특정 프로퍼티만 선택합니다. 컴포넌트의 props 타입을 정의할 때 유용합니다:

```typescript
// User에서 표시에 필요한 필드만 선택
type UserCard = Pick<User, 'name' | 'avatar'>;

function UserCardComponent({ name, avatar }: UserCard) {
  return (
    <div>
      <img src={avatar} alt={name} />
      <span>{name}</span>
    </div>
  );
}
```

### Omit&lt;T, K&gt;

특정 프로퍼티를 제외합니다. 생성 요청에서 자동 생성 필드를 제외할 때 많이 사용합니다:

```typescript
// 생성 시 id는 서버에서 자동 생성되므로 제외
type CreateUserInput = Omit<User, 'id'>;

function createUser(input: CreateUserInput) {
  return fetch('/api/users', {
    method: 'POST',
    body: JSON.stringify(input),
  });
}
```

### Record&lt;K, V&gt;

키-값 쌍의 타입을 정의합니다:

```typescript
// 상태별 색상 매핑
type Status = 'pending' | 'active' | 'closed';

const statusColors: Record<Status, string> = {
  pending: 'yellow',
  active: 'green',
  closed: 'gray',
};

// 에러 메시지 매핑
type FormField = 'name' | 'email' | 'password';
const errors: Partial<Record<FormField, string>> = {
  email: '올바른 이메일을 입력해 주세요',
};
```

### ReturnType&lt;T&gt;

함수의 반환 타입을 추출합니다:

```typescript
function getConfig() {
  return {
    apiUrl: process.env.API_URL || 'http://localhost:3000',
    debug: process.env.NODE_ENV === 'development',
    maxRetries: 3,
  };
}

// 함수의 반환값 타입을 자동 추론
type Config = ReturnType<typeof getConfig>;
// { apiUrl: string; debug: boolean; maxRetries: number }
```

### Extract&lt;T, U&gt;와 Exclude&lt;T, U&gt;

유니온 타입에서 특정 타입을 추출하거나 제외합니다:

```typescript
type Event = 'click' | 'scroll' | 'mousemove' | 'keydown' | 'keyup';

// 마우스 관련 이벤트만 추출
type MouseEvent = Extract<Event, 'click' | 'scroll' | 'mousemove'>;

// 키보드 이벤트를 제외한 나머지
type NonKeyboardEvent = Exclude<Event, 'keydown' | 'keyup'>;
```

## 실전 조합 패턴

유틸리티 타입을 조합하면 더 강력한 타입을 만들 수 있습니다:

```typescript
interface Post {
  id: number;
  title: string;
  content: string;
  author: string;
  createdAt: Date;
  updatedAt: Date;
}

// 생성: id와 날짜 필드 제외
type CreatePost = Omit<Post, 'id' | 'createdAt' | 'updatedAt'>;

// 수정: 생성 타입에서 부분 업데이트 허용
type UpdatePost = Partial<CreatePost>;

// 목록: 미리보기용으로 content 제외
type PostPreview = Omit<Post, 'content'>;
```

## 마무리

유틸리티 타입은 기존 타입을 기반으로 새로운 타입을 만드는 강력한 도구입니다. 중복을 줄이고 타입 안전성을 높이면서도 코드를 유연하게 유지할 수 있습니다. 특히 API 요청/응답 타입을 정의할 때 `Partial`, `Pick`, `Omit`의 조합은 거의 필수적으로 사용됩니다.
