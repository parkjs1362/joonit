초기에는 글 5개를 관리하는 것이 쉽습니다.  
문제는 30개를 넘어갈 때 시작됩니다.

제목 톤, 썸네일 스타일, 설명 길이가 들쭉날쭉해지면  
사이트는 금방 “임시 페이지”처럼 보입니다.

## 1) 카테고리별 커버 이미지를 먼저 고정

글마다 다른 이미지를 넣는 대신,  
카테고리 대표 커버를 기본값으로 두면 일관성이 급격히 올라갑니다.

- 개발: 기술/인터페이스 톤
- 역사: 기록/연표 톤
- 일상: 메모/루틴 톤

필요한 글만 개별 이미지를 override하는 방식이 운영에 유리합니다.

## 2) frontmatter 필드를 최소 필수로 강제

아래는 빠지면 품질이 떨어지는 최소 세트입니다.

- `title`
- `description`
- `date`
- `category`
- `tags`

특히 `description`은 카드 클릭률에 직접 영향을 줍니다.

## 3) 분량 기준을 정해 편차 줄이기

이 블로그에서는 2~4분 읽기를 기준으로 두면  
작성 속도와 완성도를 동시에 가져갈 수 있습니다.

- 너무 짧으면 맥락 부족
- 너무 길면 다음 글 발행이 밀림

## 4) 제목 규칙을 “문제 + 해결”로 통일

예시:

- “파비콘이 안 바뀌는 이유: 캐시와 버전링”
- “요청 문장이 흔들리는 이유: 추상어 줄이기”

이 구조는 검색성과 스캔 가독성 모두에서 유리합니다.

## 5) 카테고리별 기본 템플릿 하나씩 만들기

글쓰기 피로를 줄이려면 템플릿이 필요합니다.

- 개발: 문제 → 원인 → 체크리스트
- 역사: 팩트 체크 → 연표 → 해석
- 일상: 상황 → 루틴 → 오늘의 메모

매번 처음부터 쓰지 않으면 발행 주기가 안정됩니다.

## 6) 월 1회 정리 작업을 따로 둔다

콘텐츠가 쌓일수록 “생산”과 “정리”를 분리해야 합니다.

- 태그 정규화
- 오래된 설명 문장 보정
- 카테고리 오분류 수정

정리 시간을 분리하면 전체 사이트 인상이 깔끔해집니다.

## 오늘의 체크리스트

- 카테고리 커버 기본값 유지
- frontmatter 최소 세트 누락 0
- 템플릿으로 2~4분 글을 안정적으로 발행

콘텐츠 운영은 영감보다 규칙이 더 오래 갑니다.
