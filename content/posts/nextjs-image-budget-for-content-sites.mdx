콘텐츠 사이트에서 이미지는 분위기를 좌우합니다.  
문제는 보기 좋은 이미지를 넣을수록 LCP, 스크롤 안정성, 모바일 데이터 사용량이 함께 나빠지기 쉽다는 점입니다.

그래서 저는 이미지 최적화를 기술 옵션이 아니라 **예산 관리**로 봅니다.

## 1) 먼저 정해야 할 건 포맷보다 예산이다

대부분의 팀이 "WebP로 바꾸자"부터 말합니다.  
하지만 더 중요한 건 페이지 안에서 이미지를 몇 장, 어느 크기로, 어떤 우선순위로 둘지 정하는 일입니다.

제가 자주 쓰는 기본 예산은 이렇습니다.

- 리스트 카드 썸네일: 80KB 안팎
- 본문 대표 이미지: 160KB 안팎
- 첫 화면 핵심 이미지 총량: 250KB 이내
- 한 페이지 전체 이미지 합: 모바일 기준 700KB 전후

정답은 아니지만, 이 정도 기준이 있어야 과잉 사용을 빨리 감지할 수 있습니다.

## 2) 콘텐츠 사이트는 '첫 화면'에 가장 엄격해야 한다

홈이나 목록 페이지는 카드가 많아서 이미지 부채가 순식간에 커집니다.

- 첫 3개 카드만 강조하고 나머지는 더 가볍게
- 중요한 카드 외에는 `priority` 사용 최소화
- 동일한 레이아웃이면 썸네일 비율을 통일

특히 블로그 목록은 이미지 품질보다 **리듬과 일관성**이 더 중요합니다.

## 3) 예산이 깨질 때 우선 줄일 순서

1. 해상도
2. 품질 값
3. 노출 개수
4. 이미지 자체 필요성

보통은 품질만 건드리지만, 실제로는 노출 개수와 레이아웃이 더 큰 영향을 줍니다.  
"꼭 보여줘야 하나?"를 먼저 물어야 합니다.

## 4) 본문 이미지는 문맥 단위로만 둔다

긴 글에서 이미지를 남발하면 읽는 흐름이 끊깁니다.

- 섹션 전환점에서만 사용
- 설명 없이 장식용이면 삭제
- 같은 정보를 텍스트와 이미지로 중복하지 않기

콘텐츠 사이트의 이미지는 갤러리가 아니라 **이해를 돕는 장치**여야 합니다.

## 5) 운영 단계에서는 무엇을 확인하나

- 홈 최신 글 카드 6개 로딩 시 첫 화면 무게
- 모바일 4G 기준 이미지 로드 체감
- 대표 이미지 교체 후 LCP 변동
- 새 글 추가 시 목록 페이지 총 이미지 수 증가 폭

이미지 예산은 한 번 정하고 끝나지 않습니다.  
새 글이 쌓일수록 계속 감시해야 합니다.

## 체크리스트

- [ ] 홈/목록/상세별 이미지 예산을 숫자로 정했는가
- [ ] 첫 화면 핵심 이미지 총량을 따로 관리하는가
- [ ] `priority` 사용을 정말 필요한 곳만 제한했는가
- [ ] 본문 이미지는 문맥 전환점 중심으로만 배치했는가

성능은 압축 기술보다 운영 규칙에서 더 자주 무너집니다.  
이미지 예산은 디자이너와 개발자 모두를 편하게 만드는 공통 기준입니다.
