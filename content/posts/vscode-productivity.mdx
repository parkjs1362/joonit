## VS Code를 선택하는 이유

Visual Studio Code는 가볍고 빠르면서도 강력한 확장 생태계를 가진 코드 에디터입니다. 적절한 설정과 확장 프로그램을 조합하면 전문 IDE 못지않은 개발 환경을 구축할 수 있습니다.

## 필수 확장 프로그램

### 코드 품질

```
ESLint          - JavaScript/TypeScript 린팅
Prettier        - 코드 포맷팅 자동화
Error Lens      - 에러를 인라인으로 표시
```

### 개발 편의

```
GitLens         - Git 히스토리와 blame 시각화
Auto Rename Tag - HTML 태그 동시 수정
Path Intellisense - 파일 경로 자동완성
```

### UI/테마

```
One Dark Pro    - 인기 다크 테마
Material Icon   - 파일 아이콘 테마
Indent Rainbow  - 들여쓰기 수준별 색상
```

## 핵심 설정 (settings.json)

```json
{
  "editor.fontSize": 14,
  "editor.lineHeight": 1.6,
  "editor.tabSize": 2,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": "active",
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "files.autoSave": "onFocusChange",
  "explorer.confirmDelete": false,
  "terminal.integrated.fontSize": 13
}
```

핵심 포인트:
- `formatOnSave`: 저장 시 자동 포맷팅
- `codeActionsOnSave`: 저장 시 ESLint 자동 수정
- `bracketPairColorization`: 괄호 짝 색상 구분

## 생산성을 높이는 단축키

### 파일 탐색

| 단축키 | 기능 |
|--------|------|
| `Cmd+P` | 파일 빠른 열기 |
| `Cmd+Shift+P` | 명령 팔레트 |
| `Cmd+Shift+F` | 전체 파일 검색 |
| `Cmd+B` | 사이드바 토글 |

### 코드 편집

| 단축키 | 기능 |
|--------|------|
| `Alt+Up/Down` | 줄 이동 |
| `Shift+Alt+Up/Down` | 줄 복사 |
| `Cmd+D` | 같은 단어 다중 선택 |
| `Cmd+Shift+K` | 줄 삭제 |
| `Cmd+/` | 주석 토글 |
| `Cmd+Shift+L` | 동일 단어 모두 선택 |

### 멀티 커서

| 단축키 | 기능 |
|--------|------|
| `Alt+Click` | 커서 추가 |
| `Cmd+Alt+Up/Down` | 위/아래 커서 추가 |
| `Cmd+D` | 다음 동일 단어에 커서 |

## 워크스페이스 설정

프로젝트별 설정은 `.vscode/settings.json`에 저장합니다:

```json
{
  "typescript.preferences.importModuleSpecifier": "non-relative",
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "search.exclude": {
    "**/node_modules": true,
    "**/.next": true
  }
}
```

## 코드 스니펫

반복적인 코드를 스니펫으로 등록하면 생산성이 크게 향상됩니다:

```json
// .vscode/typescriptreact.code-snippets
{
  "React FC": {
    "prefix": "rfc",
    "body": [
      "interface ${1:Component}Props {",
      "  $2",
      "}",
      "",
      "export default function ${1:Component}({ $3 }: ${1:Component}Props) {",
      "  return (",
      "    <div>",
      "      $0",
      "    </div>",
      "  );",
      "}"
    ]
  }
}
```

`rfc` + Tab 키만 누르면 전체 컴포넌트 구조가 생성됩니다.

## 터미널 통합

VS Code의 내장 터미널을 적극 활용하세요:

- `Ctrl+`` `: 터미널 열기/닫기
- `Cmd+\`: 터미널 분할
- 터미널에서 작업하면서 에디터와 빠르게 전환 가능

## 마무리

VS Code의 진정한 가치는 확장성에 있습니다. 자신의 워크플로우에 맞게 설정을 커스터마이징하고, 반복 작업을 자동화하면 코딩에만 집중할 수 있는 환경을 만들 수 있습니다. 처음에 시간을 투자해 환경을 최적화하면 매일의 생산성이 누적되어 큰 차이를 만듭니다.
