SEO는 보통 '상위 노출'로만 이야기되지만, 개발자 입장에서 더 중요한 건 따로 있습니다.

> 검색엔진이 내 페이지를 "정상적인 문서"로 이해하게 만드는 것

즉, SEO는 꾸밈이 아니라 **기본 공사**에 가깝습니다.

## 1) 문서 단위 기본값

아래가 비어 있으면, 검색/공유가 어색해집니다.

- `title`
- `description`
- `openGraph`(공유 미리보기)
- `robots`(인덱싱 허용/차단)

Next.js App Router라면 페이지별로 `metadata` 또는 `generateMetadata`로 정리할 수 있습니다.

```ts
export const metadata = {
  title: 'Joonit',
  description: '개발·역사·일상 기록',
};
```

## 2) 글(포스트) 단위 메타데이터

포스트는 특히 공유에 민감합니다.

- OG 이미지가 없으면 카드가 밋밋해지고
- 제목/요약이 없으면 클릭할 이유가 사라집니다

그래서 포스트 메타는 "자동"이 중요합니다.

- 글에 `image`가 없으면 카테고리 커버로 대체
- 글에 `description`이 없으면 첫 문단/요약을 생성(선택)

## 3) 사이트 레벨 문서

AdSense나 신뢰 관점에서도 도움이 됩니다.

- `/sitemap.xml`
- `/robots.txt`
- `/privacy`
- `/terms`
- `/about`, `/contact`

여기까지가 갖춰지면, 검색엔진이나 심사 시스템이 볼 때 "살아있는 사이트"로 보이기 쉬워집니다.

## 4) 실전 체크 7가지

배포 전에 아래만 봐도 대부분 커버됩니다.

- 깨진 링크가 없는가(특히 더미 링크)
- 모바일에서 본문이 너무 작거나 넓지 않은가
- OG 이미지/제목이 정상적으로 나오는가
- 카테고리/글 목록에서 최신 글이 충분히 보이는가
- 개인정보/약관/연락처 페이지가 접근 가능한가
- `ads.txt`가 있다면 루트에 노출되는가
- 로딩이 너무 느리지 않은가(이미지 최적화)

## 결론

SEO는 "최적화"보다 "기본값"입니다.

- 문서 메타데이터
- 사이트 구조 문서
- 끊긴 링크 제거

이 3가지만 탄탄하면, 나머지는 콘텐츠가 결정합니다.
